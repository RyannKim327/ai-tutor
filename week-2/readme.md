# Week 2: Classifying Shapes

## Introduction to Shape Classification
In this lesson, we will learn how to identify and classify basic shapes. Just like in our daily lives where we recognize different objects (like how we know a ball is round and a box is square), programming allows us to do something similar with shapes.

### Real-life Analogy
Imagine you're at a toy store. You see different toys: some are round like balls, others are flat squares like board games, and some are pointy like pyramids. If someone asks you to sort these toys, you would look at their properties—like how many sides or corners they have—to decide where each toy belongs. This is exactly what we will do in our programming challenge.

### Concept Breakdown
1. **Identify Properties**: We need to consider the properties of shapes. For example:
   - Circles have no sides but can be described by their radius.
   - Squares have four equal sides and four right angles.
   - Triangles have three sides.
   - Rectangles have four sides but they can have unequal lengths.

2. **Input Characteristics**: In our program, we will gather details about the shapes. We need to know how many sides the shape has and any other defining features.

3. **Shape Classification**: After we have the properties, we’ll compare them to determine if we’ve got a circle, triangle, square, rectangle, or something else we don’t know.

4. **Output Result**: Finally, we will display what shape we identified based on the properties we analyzed.

### Implementation Inside challenge.py
To tackle this challenge, you'll create a section in your `challenge.py` file where you'll:
- Think through the properties of each shape and represent them logically.
- Write a plan that follows the steps mentioned above in the comments.
